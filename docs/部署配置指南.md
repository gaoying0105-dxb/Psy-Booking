# 部署配置指南（手把手）

按顺序做完以下 5 步，网站就能上线。全程 0 元。

---

## 第 1 步：创建 Supabase 项目（约 5 分钟）

1. 打开 https://supabase.com，用 GitHub 账号登录
2. 点 **New Project**：
   - Name：随便起，如 `xinli-booking`
   - Database Password：设一个强密码并**记下来**（一般用不到，但丢了麻烦）
   - Region：选 **Northeast Asia (Tokyo)** 或 **Southeast Asia (Singapore)**，国内访问相对快
3. 等 1-2 分钟项目初始化完成

### 1.1 运行数据库脚本

1. 左侧菜单 → **SQL Editor** → **New query**
2. 把项目里 `supabase/schema.sql` 的**全部内容**粘贴进去 → 点 **Run**
   - 看到 "Success. No rows returned" 即成功
3. 再新建一个 query，粘贴 `supabase/seed.sql` 的内容 → **Run**
   - 这一步会写入默认预约须知 + 未来 14 天的示例时段

### 1.2 获取 API 密钥

左侧菜单 → **Project Settings** → **API**，记下两个值：

| 名称 | 在哪 | 对应环境变量 |
|---|---|---|
| Project URL | `https://xxxx.supabase.co` | `NEXT_PUBLIC_SUPABASE_URL` |
| anon public key | API Keys 区域的 `anon` `public` | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |

> ⚠️ 只用 **anon** 这个 key。旁边的 `service_role` key 是超级权限密钥，**永远不要**放进前端代码或环境变量带 `NEXT_PUBLIC_` 前缀的地方。

### 1.3 关闭邮箱验证（MVP 推荐，省事）

1. 左侧菜单 → **Authentication** → **Sign In / Up** → **Email**
2. 找到 **Confirm email** 开关 → 关闭 → Save

这样用户注册后直接登录，不用去邮箱点确认链接。等正式运营再开启也行（开启后注册确认邮件免费版每小时限 2 封，正式用需配置自己的 SMTP）。

---

## 第 2 步：本地跑起来（约 3 分钟）

```bash
# 需要 Node.js 18+，没有的话去 https://nodejs.org 装 LTS 版

cd xinli-booking
cp .env.local.example .env.local
# 用编辑器打开 .env.local，填入第 1.2 步拿到的两个值

npm install
npm run dev
```

打开 http://localhost:3000 ：
1. 点「登录 / 注册」→ 用你自己的邮箱注册一个账号
2. 试着预约一个时段，跑通流程

---

## 第 3 步：把自己设为管理员（1 分钟）

回到 Supabase → **SQL Editor**，运行（邮箱换成你刚注册的）：

```sql
update public.profiles set role = 'admin' where email = 'your-email@example.com';
```

刷新网站，导航栏会出现「管理后台」入口。进去试试：确认刚才的预约、屏蔽一个时段、改一下预约须知。

---

## 第 4 步：部署到 Vercel（约 5 分钟）

### 4.1 推送代码到 GitHub

```bash
cd xinli-booking
git init
git add .
git commit -m "init: 心理咨询预约系统"
```

在 https://github.com/new 创建一个**私有**仓库（不填任何初始化选项），然后：

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

### 4.2 在 Vercel 导入

1. 打开 https://vercel.com，用 GitHub 登录
2. **Add New → Project** → 选中刚才的仓库 → Import
3. 在 **Environment Variables** 添加两条（和 .env.local 里一样）：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. 点 **Deploy**，等 1-2 分钟

完成后你会得到一个 `https://xxx.vercel.app` 的网址。以后每次 `git push`，Vercel 自动重新部署。

> 注：vercel.app 域名在国内部分网络环境下访问不稳定。如果目标用户在国内，建议之后绑定一个自己的域名（Vercel 项目 Settings → Domains，免费）。

---

## 第 5 步：配置保活，防止数据库休眠（约 3 分钟）

Supabase 免费版 7 天无请求会暂停数据库。项目里已带好保活脚本
`.github/workflows/keep-alive.yml`（每周一、四自动 ping 一次），只需配置密钥：

1. GitHub 仓库页 → **Settings** → **Secrets and variables** → **Actions**
2. **New repository secret**，添加两条：
   - `SUPABASE_URL` = 你的 Project URL
   - `SUPABASE_ANON_KEY` = 你的 anon key
3. 验证：仓库 → **Actions** 标签页 → 选 "Supabase Keep Alive" → **Run workflow** 手动跑一次，绿色 ✓ 即成功

## 完成 ✅ 上线后的日常

| 事项 | 在哪做 |
|---|---|
| 排每周可约时段 | 网站 /admin/slots（有"按模板添加整天"按钮）|
| 确认/拒绝预约 | 网站 /admin |
| 临时有事屏蔽时段 | /admin/slots 点「屏蔽」|
| 用户不足48h想取消 | 联系你之后，你在 /admin 替 TA 取消 |
| 标记失约 | /admin 对已确认的过期预约点「标记失约」，累计2次自动禁约 |
| 修改预约须知 | /admin/settings |
| 修改首页咨询师介绍 | 改代码 `src/app/page.tsx` 里的文案，push 即自动部署 |

## 常见问题

**Q: 注册时提示发不出邮件 / 收不到确认邮件？**
按第 1.3 步关闭 Confirm email 即可。

**Q: 部署后页面 500 错误？**
九成是 Vercel 环境变量没填或填错。Vercel 项目 → Settings → Environment Variables 检查，改完要 Redeploy。

**Q: 想换预约时段模板（比如加晚上场）？**
改 `src/app/admin/slots/page.tsx` 顶部的 `TEMPLATE` 数组。

**Q: 以后想加邮件/短信通知？**
推荐 Resend（免费 3000 封/月）+ Supabase Edge Functions 或数据库 Webhook，预约状态变化时触发。这是设计好的扩展点，需要时可以再迭代。
